# แผนการดำเนินงาน: sdd-https-vpn-ciphersuite-th

> เวอร์ชัน: 1.1

> อัปเดตล่าสุด: 2026-05-11

## สรุป (Summary)

แผนการดำเนินงานจะมุ่งเน้นการสร้างระบบไฮบริดที่ยืดหยุ่น โดยแบ่งการพัฒนาเป็นระดับตามความเสี่ยงและการใช้งาน (Use-case based deployment)

## รายละเอียดงาน (Task Breakdown)

### เฟส 1: โครงสร้างพื้นฐานและ KEM Profiles (Foundation & KEM)

#### งาน 1.1: สร้างโครงสร้างโฟลเดอร์และนิยาม Named Groups
- **คำอธิบาย**: นิยามค่าคงที่สำหรับ X25519_MLKEM768 และ P384_MLKEM1024
- **ไฟล์**: `crypto/th/tls/named_groups.go`, `crypto/th/provider.go`
- **ความซับซ้อน**: ต่ำ

#### งาน 1.2: พัฒนา Hybrid Key Exchange Logic
- **คำอธิบาย**: รวมการทำงานระหว่าง X25519/P-256 และ ML-KEM-768/1024
- **ไฟล์**: `crypto/th/kem.go`
- **ความซับซ้อน**: สูง

### เฟส 2: ระบบการลงนามและใบรับรอง (Signatures & Certificates)

#### งาน 2.1: พัฒนา Hybrid Signature Provider (ML-DSA-65)
- **คำอธิบาย**: รองรับการลงนามแบบไฮบริดสำหรับทราฟฟิกทั่วไปและ API
- **ไฟล์**: `crypto/th/signature.go`
- **ความซับซ้อน**: ปานกลาง

#### งาน 2.2: พัฒนา SLH-DSA สำหรับ Trust Anchors
- **คำอธิบาย**: พัฒนาโมดูลสำหรับการลงนามเฟิร์มแวร์และ Root Manifest
- **ไฟล์**: `crypto/th/slh_dsa.go`
- **ความซับซ้อน**: สูง (เนื่องจากขนาดและความซับซ้อนของอัลกอริทึม)

### เฟส 3: ระบบสำรองและการวิเคราะห์ช่องว่าง (Backup & Gap Analysis)

#### งาน 3.1: รวม HQC เป็น Backup KEM
- **คำอธิบาย**: เพิ่ม HQC เข้าในรายการ KEM ที่รองรับเพื่อเป็นระบบสำรอง
- **ความซับซ้อน**: ปานกลาง

#### งาน 3.2: วิจัยและเตรียมการสำหรับ Falcon
- **คำอธิบาย**: เตรียมโครงสร้างไฟล์และ Interface เพื่อรองรับ Falcon ในอนาคต
- **ความซับซ้อน**: ต่ำ

### เฟส 4: การตรวจสอบและความเข้ากันได้ (Verification & Compatibility)

#### งาน 4.1: ทดสอบประสิทธิภาพ (Performance Test)
- **คำอธิบาย**: เปรียบเทียบ Latency ระหว่าง ML-KEM-768 และ 1024 ในสถานการณ์จริง

#### งาน 4.2: ทดสอบขอบเขต (Edge Case Test)
- **คำอธิบาย**: ทดสอบการจัดการแพ็กเก็ตขนาดใหญ่ของ SLH-DSA ในระบบเครือข่าย
